## 🚀 우선순위별 개선 제안

### 🔴 High Priority (즉시 구현 권장)

#### 1. 회원가입 & 인증 시스템
**목적**: 데이터 영구 보존 및 멀티 디바이스 동기화

**구현 방안**:
```typescript
// Firebase Authentication 활용
- Google 소셜 로그인
- 이메일/비밀번호 로그인
- 익명 로그인 → 나중에 계정 연동 옵션

// 데이터 구조
interface User {
  uid: string;
  email: string;
  displayName: string;
  photoURL: string;
  createdAt: Date;
  subscription: 'free' | 'pro';
}
```

**UX 플로우**:
1. 앱 첫 실행 → "게스트로 시작" or "로그인"
2. 게스트 모드: localStorage 사용
3. 로그인 시: Firestore로 데이터 마이그레이션
4. 설정 페이지에서 언제든 계정 연동 가능

**예상 개발 시간**: 2-3일

## 🔧 기술적 개선 사항

### 1. 성능 최적화
```typescript
// React.memo로 불필요한 리렌더링 방지
const TaskCard = React.memo(({ task }) => { ... });

// useMemo로 복잡한 계산 캐싱
const totalFocusTime = useMemo(() => 
  tasks.reduce((sum, t) => sum + t.totalSeconds, 0),
  [tasks]
);

// 디바운싱으로 localStorage 쓰기 최적화
const debouncedSave = debounce((data) => {
  localStorage.setItem('userStats', JSON.stringify(data));
}, 1000);
```

### 2. 에러 처리
```typescript
// localStorage 접근 실패 대비
try {
  const data = localStorage.getItem('userStats');
  // ...
} catch (error) {
  console.error('Storage access failed:', error);
  // Fallback to in-memory state
}

// 타이머 동기화 오류 복구
if (Math.abs(serverTime - clientTime) > 5000) {
  syncWithServer();
}
```
